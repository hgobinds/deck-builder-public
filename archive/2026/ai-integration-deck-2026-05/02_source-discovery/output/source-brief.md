# Source Brief

## Approved Sources

### S1 — Consultant Tooling Proposal (Internal)

**Core thesis (usable verbatim):**
> "The goal is not to automate the consultant out of the process. The goal is to remove repeated setup, forgotten context, inconsistent review, and weak handoff so the consultant can spend more time on judgment, diagnosis, and client trust."

**Consultant operating loop (7 stages):**
Intake + Discovery → Scope + Proposal → Delivery Workspace → Review + Client Comms → Handoff + Case Study

**Three pillars:** Client Setup / Delivery Work / Practice Growth

**15 AI tools across 3 phases:**
- Phase 1 (Core): Workspace Generator, Intake Brief Builder, Scope Agreement Builder, Meeting Notes Processor, Review Gate, Client Comms Drafter, Handoff Package Builder
- Phase 2 (Control): Change Request Detector, Workspace Auditor, Decision Log Builder, Command Center
- Phase 3 (Growth): Proposal Builder, Case Study Anonymizer, Deliverable Builder, Practice Analytics

**Design principle:** Each tool does one job. AI handles setup, routing, formatting, logging — consultant handles judgment, diagnosis, trust.

**Success criteria:** Consultants spend more time on judgment and client relationships, not admin.

## Claims to Avoid

- Do not claim specific productivity percentages or ROI figures — none are cited in the source
- Do not imply all 15 tools are already built or deployed — this is a proposal document
- Do not overstate the scope; frame as a plan and direction, not a completed rollout
