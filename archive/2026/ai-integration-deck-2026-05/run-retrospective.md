# Run Retrospective: AI Integration Plan

## Built

2026-05-07. Presented 2026-05-08.

## What Worked

- Source-first approach: the internal proposal PDF provided the core thesis, operating loop, and tool list — no external research needed
- ASCII diagrams gave the deck a distinct look without requiring design assets
- Humaniser skill on speaker notes produced noticeably cleaner talk tracks
- Live demo slide (slide 10) is a strong meta-moment — the deck was built by the toolkit it presents about
- Discussion-first process at every stage felt right — user stays in the deciding chair

## What Did Not Work

- Intake started with a one-line brief request and did not give the user a natural point to add more context. User had to prompt for this. Fixed mid-session by updating all stage CONTEXT.md files to require discussion before producing output — but intake should catch this at the start, not after.

## Process Changes Made During This Run

- All stage CONTEXT.md files updated to require "discuss first" before producing output
- Source discovery updated to ask about user documents before running any searches
- Humaniser skill added as a required step for speaker notes in build stage
- File naming convention changed from `deck-v1.pptx` to `YYMMDD_Presentation_Name.pptx`
- build_deck.py extracted into a reusable deck builder skill

## Lessons for Next AI Update Deck

- Start intake by asking for as much context as the user wants to give, not just a one-liner
- ASCII diagrams from source documents are a fast, distinctive visual approach — worth defaulting to for internal decks
- Humaniser on speaker notes is worth the extra step — the difference is noticeable
- The live demo / meta-moment slide is a strong closer for any deck about a tool being used to build the deck itself
