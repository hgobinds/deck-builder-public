# Reusable Patterns

Use this file for durable deck intelligence discovered across projects.

## Strong Openings

- Address the fear directly (slide 3, AI integration deck): name the concern the audience has in the room before they can ask it. Clears the air and earns attention for the rest of the deck.

## Strong Closing Structures

- Meta-moment closer: if the deck was built using the tool it presents about, say so explicitly as the second-to-last slide. Makes the argument concrete without a case study. Used in AI integration deck slide 10.

## Slide Patterns That Worked

- AI integration deck example: see `_references/approved-prior-decks/ai-integration-presentation-example.pptx` for approved prior-deck structure and pacing.
- Two-layer split (operating vs judgment): strong framing for any AI enablement deck. Gives the audience a clear mental model without making them feel threatened. See `archive/2026/ai-integration-deck-2026-05/` slide 4.

## Visual Motifs That Worked

- open template: see `_templates/open-template.pptx` for reusable template-based layouts.
- ASCII diagrams on internal decks: fast to produce, distinct look, no design assets needed. Render in Courier New 10pt on a light grey (`#F2F2F2`) background. Start from `_skills/deck-boilerplate.py`. Works well for flow diagrams, tool lists, phase roadmaps.

## Source Patterns

- One strong internal document can be enough: if the user has authored a proposal or internal report, start there before running any web searches. The source may contain the thesis, structure, and examples already. Used in AI integration deck — no external sources needed.

## Common Failure Modes

- Treating a one-line brief as complete: always invite more context before drafting intake. Users often have more to share but won't offer it unless asked.
- Jumping to web search before asking about user documents: always ask what the user has first.
