# Proposal Deck Outline

## Purpose

Turn scope, approach, and investment into a decision-ready package.

## Default Flow

1. Proposal title.
2. Client context.
3. Objectives.
4. Recommended approach.
5. Workstreams or phases.
6. Timeline.
7. Team.
8. Deliverables.
9. Investment.
10. Assumptions and dependencies.
11. Approval path / next steps.

## Common Failure Modes

- Unclear scope.
- Hidden assumptions.
- Vague deliverables.
- No acceptance criteria.

## Design Defaults

- Professional, precise, and confidence-building.
- Use scope tables, phase diagrams, timelines, and decision summaries.
