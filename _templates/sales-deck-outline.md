# Sales Deck Outline

## Purpose

Help a buyer understand problem fit, value, proof, implementation path, and next step.

## Default Flow

1. Buyer-specific title.
2. Situation / context.
3. Problem or cost of status quo.
4. Insight.
5. Recommended solution.
6. How it works.
7. Proof / case study.
8. Implementation path.
9. Commercial model or package.
10. Next step.

## Common Failure Modes

- Company-first opening.
- Generic claims.
- Weak buyer-specific relevance.
- Unclear next action.

## Design Defaults

- Clear, practical, trust-building.
- Prioritize diagrams, comparisons, and proof.
- Use buyer language over internal product language.
