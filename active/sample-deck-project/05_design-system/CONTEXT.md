# Stage 05: Design System

## What This Stage Does

Define the visual system for this deck before building slides.

## What To Load

| Situation | Load | Skip These | Output | Trigger |
|---|---|---|---|---|
| Default | `../04_storyboard/output/slide-storyboard.md`, `../../../_config/brand-guide.md`, `../../../_config/visual-constraints.md`, relevant `_templates/` file | Unrelated brand systems, old drafts, raw research | `output/design-plan.md` | Storyboard is approved |

## Process

1. **Discuss first.** Before choosing a direction, ask the user:
   - Do you have a preferred visual style or existing template to use?
   - Are there brand constraints or colors to follow?
   - Are there slides with specific visual ideas you already have in mind?
2. Get agreement on the visual direction before writing the design plan.
3. Use user-specified visual direction when provided.
4. If none is specified, start from the relevant configured template.
5. Define tone, color roles, typography, motif, layout system, asset needs, and constraints.

## Acceptance Criteria

- Template or visual system is selected.
- Color, type, layout, motif, asset needs, and constraints are documented.
- The design plan explains when it follows a configured template and when it intentionally departs from one.

## Approval Gate

Review required by: deck owner or brand reviewer.
Approved to proceed?: yes/no.

## What Not To Do

- Do not choose a visual direction without discussing it with the user first.
- Do not invent a new design system when a configured template or brand pattern fits.
- Do not default to generic blue.
- Do not make a text-only visual plan unless the deck type explicitly calls for it.
